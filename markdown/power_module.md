# 电源管理模块设计与实现

## 概述

系统采用树莓派专用 UPS 电源模块为 Raspberry Pi 4B 供电，该模块通过 40-pin GPIO 顶针直接与主板连接，支持边充边放（Pass-through），适用于无人值守的便携部署场景。为保障系统在电池供电条件下的可靠运行，本节从硬件接口、软件驱动、低电量保护及状态显示四个层面描述完整的电源管理实现。

---

## 硬件接口

### 电源模块规格

| 参数 | 值 |
|---|---|
| 供电方式 | 40-pin 顶针（GPIO 排针直连） |
| 主输出（OUTPUT2） | 5V / 4A，UPS 稳定输出 |
| 充电保护 | 过充、过热保护 |
| 放电保护 | 过热、过流、欠压（默认 3.0V 关断）、短路保护 |
| 串口参数 | 3.3V TTL，115200 baud，8N1 |
| 串口数据上报 | 每 2 秒自动上报 BATTERY / OUTPUT1 / OUTPUT2 / DCINPUT 电压 |

### 串口引脚分配

电源模块的串口通信复用树莓派硬件 UART（mini UART），引脚分配如下：

| 信号 | BCM 编号 | 物理 Pin |
|---|---|---|
| UART TX（Pi → 模块） | GPIO14 | Pin 8 |
| UART RX（模块 → Pi） | GPIO15 | Pin 10 |

在引脚分配设计中，GPIO15（UART RX）与原按键 1 存在复用冲突。由于 UART 硬件初始化后会接管 GPIO15 的引脚模式，lgpio 对该引脚的 GPIO 输入配置会导致 UART 接收失效。为此，将按键 1 从 GPIO15（Pin 10）迁移至 GPIO4（Pin 7），彻底消除冲突。

树莓派系统配置要求：

```
# /boot/firmware/config.txt
enable_uart=1
```

`/boot/firmware/cmdline.txt` 中不得包含 `console=serial0`，以禁用串口控制台，确保 `/dev/ttyS0` 可被用户程序独占使用。

## 串口驱动实现

### 模块设计

串口驱动封装在 `src/power/serial_reader.py` 的 `PowerReader` 类中，以 asyncio task 形式运行，不阻塞主事件循环。

**核心设计要点：**

- 串口读取采用 `run_in_executor` 将阻塞的 `serial.readline()` 委托至线程池，保持 asyncio 事件循环畅通
- 设置 `timeout=0.5s`，使读取线程每 0.5 秒返回一次，以便在取消 task 时能及时退出
- 电压数据与运行时长数据在同一串口流中混合上报，通过格式特征区分解析

### 数据解析

电源模块串口输出两类数据：

**电压上报（每 2 秒自动）：**
```
BATTERY:3.71V
OUTPUT1:5.19V
OUTPUT2:5.21V
DCINPUT:4.78V
```

**运行时长查询（主动发送 `ATE\r\n` 指令触发）：**
```
0天:2时:15分:30秒
```

解析策略：优先通过正则表达式检测中文时间格式（含"天"字符）识别运行时长响应，其余行按 `KEY:VALUE` 结构解析电压值。

### 电量百分比估算

由于该电源模块不支持电流采样，无法直接计算功率或精确剩余电量。采用锂电池开路电压线性映射方法进行估算：

$$\text{SOC}(\%) = \frac{V_{\text{bat}} - V_{\min}}{V_{\max} - V_{\min}} \times 100$$

其中 $V_{\min} = 3.0\text{V}$（截止电压），$V_{\max} = 4.2\text{V}$（满充电压），结果钳位于 $[0, 100]$。

该方法精度有限（锂电池放电曲线非线性），但在无电流传感器的条件下可提供直观的电量参考。

### 运行时长查询

`PowerReader` 在启动后立即发送一次 `ATE` 指令，此后每 60 秒刷新一次。`serial.write()` 为非阻塞操作，可在读取线程阻塞于 `readline()` 时安全调用，无需额外加锁。

## 低电量保护

### 设计动机

电源模块默认在电池电压降至 3.0V 时通过硬件 MOSFET 切断输出，Pi 将瞬间断电。若系统此时正在写入文件（日志、推理结果等），SD 卡文件系统存在损坏风险。软件层面的安全关机机制在硬件断电前完成文件 flush 与系统下电，可有效规避此风险。

### 保护阈值

| 阶段 | 电压阈值 | 行为 |
|---|---|---|
| 低电量警告 | ≤ 3.4V | OLED 副屏强制切换至电源页；电量图标闪烁提醒 |
| 安全关机 | ≤ 3.2V | 停止采集 → OLED 提示 → 3 秒延迟 → `shutdown -h now` |
| 硬件断电（模块默认） | ≤ 3.0V | 硬件切断输出（软件关机提供 0.2V 余量） |

充电检测：若 DCINPUT 电压 > 4.0V，判定为 DC 接入充电状态，跳过所有保护逻辑。

### 实现逻辑

保护逻辑集成于 `sysinfo_updater()` 协程，每秒执行一次检查。触发安全关机时，通过 `_exit = True` 通知主循环退出，LED 关闭，OLED 显示提示信息，等待 3 秒后执行系统关机命令，确保 Python 日志及文件系统缓冲区完成 flush。

## OLED 副屏电源页

### 显示布局（128×32）

```
Bat:  73.5%          ⚡  📶    ← 低电量时闪电图标 1Hz 闪烁
Vol:   3.71V
Up: 02:15:30                   ← 运行时长，超过 1 天显示 Xd XX:XX:XX
```

### 低电量闪电图标

设计为 6×8 像素位图，绘制于 OLED 右上角（x=100, y=0），位于 WiFi 图标左侧。闪烁效果由 `sysinfo_updater` 每 tick（1 秒）取反 `lightning_on` 标志实现，无需独立线程。图标仅在低电量且未充电时显示，正常状态下该区域为空。
